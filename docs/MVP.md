# Research Vault — MVP (зафиксировано)

## Типы материалов (VaultItem.type)

| Тип        | Описание                          |
| ---------- | --------------------------------- |
| `link`     | URL с автопарсингом метаданных   |
| `note`     | Текстовая заметка (markdown/ plain) |
| `file`     | Загруженный файл (метаданные + storage path) |
| `snippet`  | Фрагмент кода/текста              |
| `video`    | Ссылка на видео (как link + тип)  |

## Пользовательские сценарии (v1)

1. Регистрация / вход.
2. Создать материал: ссылка, заметка, файл, snippet, видео-ссылка.
3. Просмотр списка с фильтрами: тип, тег, коллекция, поиск.
4. Детальная карточка: редактирование, архив, удаление.
5. Теги: привязка к материалу, создание на лету.
6. Коллекции: папки, добавление материалов.
7. Статусы обработки для ссылок/файлов: `queued` → `processing` → `done` | `failed`.

## Минимальный набор экранов

- `/login`, `/register`
- `/` — дашборд: список материалов, поиск, фильтры, кнопка «Добавить».
- `/items/:id` — деталь материала, действия, AI-блок (если настроен ключ).
- `/collections` — список коллекций, создание.
- `/collections/:id` — материалы в коллекции.

## AI (первая итерация, опционально по `OPENAI_API_KEY`)

- Саммари по тексту/извлечённому контенту.
- Предложение тегов.
- Ответы по выбранным материалам (RAG-лайт: подстановка чанков в промпт).

## Вне scope для самого первого релиза

- Браузерное расширение, OCR, vector DB, мультитенант SaaS, биллинг.
